\begin{DoxyRemark}{Remarks}

\end{DoxyRemark}
\begin{DoxyParagraph}{}
The SHA3, SHAKE, c\+SHAKE, and KMAC implementations all share two forms of api\+: short-\/form and long-\/form. ~\newline
The short-\/form api, which initializes the state, processes a message, and finalizes by producing output, all in a single function call, for example; hkds\+\_\+sha3\+\_\+compute512(uint8\+\_\+t\texorpdfstring{$\ast$}{*} output, const uint8\+\_\+t\texorpdfstring{$\ast$}{*} message, size\+\_\+t msglen), the entire message array is processed and the hash code is written to the output array. ~\newline
The long-\/form api uses an initialization call to prepare the state, a blockupdate call if the message is longer than a single message block, and the finalize call, which finalizes the state and generates a hash, mac-\/code, or an array of pseudo-\/random. ~\newline
Each of the function families (SHA3, SHAKE, KMAC), have a corresponding set of reference constants associated with that member, example; SHAKE\+\_\+256\+\_\+\+KEY is the minimum expected SHAKE-\/256 key size in bytes, HKDS\+\_\+\+KMAC\+\_\+512\+\_\+\+MAC\+\_\+\+SIZE is the minimum size of the KMAC-\/512 output mac-\/code output array, and HKDS\+\_\+\+KECCAK\+\_\+512\+\_\+\+RATE is the SHA3-\/512 message absorption rate.
\end{DoxyParagraph}
